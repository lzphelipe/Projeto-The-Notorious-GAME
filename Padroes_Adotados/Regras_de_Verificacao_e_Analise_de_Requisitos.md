# Regras de Verificação e Análise de Requisitos

Este documento define os critérios e padrões adotados pela equipe para a verificação e análise dos requisitos do projeto **The Notorious GAME**, conforme solicitado na disciplina de Engenharia de Software (GCC188).

---

## Nomenclatura dos Requisitos
As nomenclaturas adotadas para os requisitos serão: 

- **RF**: Requisitos Funcionais, que definem o que o software deverá fornecer e seu comportamento durante a interação com o ambiente externo ao software que está sendo produzido.  

  Exemplo: RF01 - O sistema deve permitir o cadastro de novos jogos.

  Exemplo: RF02 - O sistema deve permitir o cadastro de novos clientes.

  Exemplo: RF03 - O sistema deve permitir o cadastro de novas vendas.

- **RNF**: Requisitos Não Funcionais, que restringem a forma como os requisitos funcionais fornecerão seus serviços e se comportarão durante a execução do software que está sendo produzido.
  
  Exemplo: RNF01 - O sistema deve responder às requisições em até 3 segundos.

  Exemplo: RNF02 - O sistema deve ser capaz de suportar até 100 usuários concorrentes.

  Exemplo: RNF03 - O sistema deve garantir a segurança dos dados, com autenticação e autorização.

---

## Regras de Verificação e Análise


1. **Clareza e Completude:**  
   Cada requisito deve ser redigido de forma clara, sem ambiguidades e conter todas as informações necessárias para sua implementação.

2. **Consistência:**  
   Os requisitos não devem entrar em conflito entre si. A equipe fará revisões cruzadas antes de cada entrega.

3. **Validação:**  
   Antes da implementação, os requisitos serão revisados e validados pelo grupo, garantindo que atendam às necessidades do sistema.
